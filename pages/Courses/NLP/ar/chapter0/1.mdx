## استخدام دفتر ملاحظات Google Colab [[using-a-google-colab-notebook]]

يعد استخدام دفتر ملاحظات Colab أسهل إعداد ممكن؛ قم بتشغيل دفتر ملاحظات في متصفحك وابدأ في الترميز مباشرة!

إذا لم تكن على دراية بـ Colab، نوصي بالبدء بمتابعة [المقدمة](https://colab.research.google.com/notebooks/intro.ipynb). يسمح لك Colab باستخدام بعض أجهزة التسريع، مثل وحدات معالجة الرسوميات (GPUs) أو وحدات معالجة الدقة (TPUs)، وهو مجاني للأحمال العمل الأصغر.

بمجرد أن تشعر بالراحة في التنقل في Colab، قم بإنشاء دفتر ملاحظات جديد وابدأ في الإعداد:

الخطوة التالية هي تثبيت المكتبات التي سنستخدمها في هذه الدورة. سنستخدم `pip` للتثبيت، وهو مدير الحزم لـ Python. في دفاتر الملاحظات، يمكنك تشغيل الأوامر النظامية عن طريق وضعها قبلها بالرمز `!`، لذا يمكنك تثبيت مكتبة Hugging Face Transformers كما يلي:

```
!pip install transformers
```

يمكنك التأكد من تثبيت الحزمة بشكل صحيح عن طريق استيرادها داخل وقت تشغيل Python الخاص بك:

```

```

هذا يقوم بتثبيت إصدار خفيف جدًا من مكتبة Hugging Face Transformers. على وجه الخصوص، لا يتم تثبيت أي أطر عمل للتعلم الآلي (مثل PyTorch أو TensorFlow). نظرًا لأننا سنستخدم العديد من الميزات المختلفة للمكتبة، فإننا نوصي بتثبيت الإصدار التجريبي، والذي يأتي مع جميع التبعيات المطلوبة لأي حالة استخدام يمكن تخيلها تقريبًا:

```

```

## استخدام بيئة افتراضية بايثون [[using-a-python-virtual-environment]]

إذا كنت تفضل استخدام بيئة افتراضية بايثون، فإن الخطوة الأولى هي تثبيت بايثون على نظامك. نوصي باتباع [هذا الدليل](https://realpython.com/installing-python/) للبدء.

بمجرد تثبيت بايثون، يجب أن تكون قادرًا على تشغيل أوامر بايثون في طرفيتك. يمكنك البدء بتشغيل الأمر التالي للتأكد من تثبيته بشكل صحيح قبل المتابعة إلى الخطوات التالية: `python --version`. يجب أن يقوم هذا بطباعة إصدار بايثون المتاح الآن على نظامك.

عند تشغيل أمر بايثون في طرفيتك، مثل `python --version`، يجب أن تفكر في البرنامج الذي يشغل أمرك على أنه برنامج "بايثون الرئيسي" على نظامك. نوصي بإبقاء هذا التثبيت الرئيسي خاليًا من أي حزم، واستخدامه لإنشاء بيئات منفصلة لكل تطبيق تعمل عليه - بهذه الطريقة، يمكن لكل تطبيق أن يكون له تبعياته وحزم الخاصة، ولن تحتاج إلى القلق بشأن مشكلات التوافق المحتملة مع التطبيقات الأخرى.

في بايثون، يتم ذلك باستخدام [*البيئات الافتراضية*](https://docs.python.org/3/tutorial/venv.html)، وهي أشجار دلائل ذاتية الاحتواء تحتوي كل منها على تثبيت بايثون بإصدار بايثون معين جنبًا إلى جنب مع جميع الحزم التي يحتاجها التطبيق. يمكن إنشاء مثل هذه البيئة الافتراضية باستخدام عدد من الأدوات المختلفة، ولكننا سنستخدم حزمة بايثون الرسمية لهذا الغرض، والتي تسمى [`venv`](https://docs.python.org/3/library/venv.html#module-venv).

أولاً، قم بإنشاء الدليل الذي تريد أن يعيش فيه تطبيقك - على سبيل المثال، قد ترغب في إنشاء دليل جديد يسمى *transformers-course* في جذر دليل الصفحة الرئيسية الخاصة بك:

```
mkdir ~/transformers-course
cd ~/transformers-course
```

من داخل هذا الدليل، قم بإنشاء بيئة افتراضية باستخدام وحدة بايثون `venv`:

```
python -m venv .env
```

يجب أن يكون لديك الآن دليل يسمى *.env* في مجلدك الفارغ:

```
ls -a
```

```out
.      ..    .env
```

يمكنك الدخول والخروج من بيئتك الافتراضية باستخدام نصي `activate` و`deactivate`:

```
# تنشيط البيئة الافتراضية
source .env/bin/activate

# إلغاء تنشيط البيئة الافتراضية
deactivate
```

يمكنك التأكد من تنشيط البيئة عن طريق تشغيل الأمر `which python`: إذا أشار إلى البيئة الافتراضية، فهذا يعني أنك قمت بتنشيطها بنجاح!

```
which python
```

```out
/home/<user>/transformers-course/.env/bin/python
```

### تثبيت التبعيات [[installing-dependencies]]

كما هو الحال في القسم السابق حول استخدام مثيلات Google Colab، فأنت الآن بحاجة إلى تثبيت الحزم المطلوبة للاستمرار. مرة أخرى، يمكنك تثبيت الإصدار التجريبي من مكتبة Hugging Face Transformers باستخدام مدير الحزم `pip`:

```
pip install "transformers[sentencepiece]"
```
